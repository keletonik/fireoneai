## Quick Start

### Option 1: Use Replit Agent (Recommended)

1. Open your FyreOne project in Replit
2. Open `REPLIT_AGENT_PROMPT.md`
3. Copy the entire contents
4. Paste into Replit Agent
5. Let it implement phase by phase

### Option 2: Manual Implementation

If you prefer to implement manually:

1. **Add database schema** - Copy contents of `schema-additions.ts` to your Drizzle schema
2. **Run migrations** - `npx drizzle-kit generate && npx drizzle-kit migrate`
3. **Add components** - Copy the `components/` folder to `client/src/components/`
4. **Add utilities** - Copy `utils/export.ts` to `client/src/utils/`
5. **Wire up routes** - Add API routes as described in the prompt
6. **Update layout** - Wrap your app in `SidebarProvider` and add `<Sidebar />`

## Component Usage

### Sidebar

```tsx
// In your root layout or App.tsx
import { SidebarProvider, Sidebar, SidebarTrigger } from '@/components/Sidebar';

function App() {
  return (
    <SidebarProvider>
      <div className="flex h-screen">
        <Sidebar />
        <main className="flex-1">
          {/* Your header with SidebarTrigger for mobile */}
          <header className="flex items-center p-4 border-b">
            <SidebarTrigger />
            <span className="font-bold">FYREONE</span>
          </header>
          {/* Rest of your app */}
        </main>
      </div>
    </SidebarProvider>
  );
}
```

### Message Actions

```tsx
// In your chat message component
import { MessageActions } from '@/components/MessageActions';

function AssistantMessage({ message, conversationId, index }) {
  return (
    <div className="group">
      <div className="prose">{message.content}</div>
      <MessageActions
        content={message.content}
        conversationId={conversationId}
        messageIndex={index}
      />
    </div>
  );
}
```

## Features Overview

### 1. Collapsible Sidebar
- Desktop: Always visible, can collapse to icons-only (56px)
- Mobile: Slide-in overlay with dark backdrop
- Sections: Favorites, Recent, Projects, Notebooks
- Right-click context menus on conversations
- Drag-and-drop to organize (enhancement)

### 2. Projects System
- Create folders to organize conversations
- Each project can have a knowledge base
- Move conversations between projects
- Color-coded project icons

### 3. Notebooks & Snippets
- Save specific response blocks (not whole conversations)
- Organize into named notebooks
- Add tags and notes to snippets
- Export entire notebooks as markdown
- Use as knowledge base for RAG

### 4. Export Functionality
- Export conversations as .md files
- Export as PDF (optional enhancement)
- Copy formatted content to clipboard
- Proper markdown formatting preserved

### 5. Message Actions
- Thumbs up/down for feedback
- One-click copy to clipboard
- Save to notebook button
- Visible on hover (desktop) or always (mobile)

## API Endpoints to Implement

```
# Projects
GET    /api/projects
POST   /api/projects
PATCH  /api/projects/:id
DELETE /api/projects/:id

# Conversations (updates)
PATCH  /api/conversations/:id
GET    /api/conversations/search?q=

# Notebooks
GET    /api/notebooks
POST   /api/notebooks
PATCH  /api/notebooks/:id
DELETE /api/notebooks/:id
GET    /api/notebooks/:id/snippets

# Snippets
POST   /api/snippets
PATCH  /api/snippets/:id
DELETE /api/snippets/:id
GET    /api/snippets/tags

# Export
GET    /api/conversations/:id/export?format=md
GET    /api/notebooks/:id/export?format=md
```
